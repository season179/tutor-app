What changed & why
1Photo demoted. The 17rem preview that ate ~650 px is gone. Direction A → ~84px thumbnail; Direction B → ~120px rail you can actually read the problem region in. Both zoom to full size in a dialog (zoom badge always visible — kids on tablets have no hover).
2Question is the hero. First thing after a one-line header, larger type, editable, capped at ~68ch.
3Six labels → one. Dropped "Problem context / Add the page…", "Step 1 · Confirm the question", "Upload a photo…", "Problem image", "Question", and "Review and edit if needed." A single Problem eyebrow carries it.
4No triple status. The header badge already says "Question extracted…" (commit 4f1cce8) — so the happy-path pin shows no status chip at all; only real problems (Partial / No question / Not a problem) surface, as a chip + one inline note.
5Upload sized to its label, not the full bar; becomes "Replace" after upload.
6Confirm folds the pin to one line — but leads with the goal, never a truncated problem body, so a child can't misread which thing to solve.
7"Ask about this image" is gone. The pin only does upload → review → confirm. Asking moves to a unified composer at the bottom: type or tap the mic, every turn. The confirmed photo is just context — asking about it was never a separate mode.
